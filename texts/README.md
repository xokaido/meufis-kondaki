# ტექსტები / Texts

თითო ფაილი = თითო ტექსტი აპლიკაციაში. ფაილის სახელია `<რიგი>-<id>.md` —
რიცხვითი პრეფიქსი მხოლოდ თანმიმდევრობას განსაზღვრავს ბიბლიოთეკაში.
ნომრებს შორის დატოვებულია ადგილი, რომ ახალი ტექსტი ჩასვათ გადარქმევის
გარეშე; გადანომრვაც უსაფრთხოა — მუდმივი გასაღები `id`-ია და არა ფაილის სახელი.

## ახალი ტექსტის დამატება

1. შექმენით `texts/<რიგი>-<id>.md` (id — ლათინური, მოკლე, უნიკალური)
2. ზევით ჩაწერეთ frontmatter (ქვემოთ), ქვემოთ — ტექსტი
3. გაუშვით `npm run build` — მეტი არაფერია საჭირო: ტექსტი თავად გამოჩნდება
   თავის კატეგორიაში, ძიებაშიც მოხვდება

## Frontmatter

```
---
id: vespers                 ← უნიკალური ლათინური იდენტიფიკატორი (URL-შიც ჩანს)
name: მწუხრი               ← სათაური ბარათზე
subtitle: წესი და განგება…  ← ქვესათაური ბარათზე
category: services          ← services | rites | prayers
mode: hybrid                ← (არასავალდებულო) იხ. „რეჟიმები" ქვემოთ
skipTitle: true             ← (არასავალდებულო) პირველი აბზაცი სათაურია — გამოტოვდეს
landmark: ფრაზა | იარლიყი   ← (არასავალდებულო, მეორდება) სარჩევის ანკერი:
---                            პირველი ბლოკი, რომელიც „ფრაზას" შეიცავს,
                               სარჩევში „იარლიყით" გამოჩნდება
```

## ტექსტის ფორმატი

- `**მოკლე გამუქებული სტრიქონი**` → განყოფილების სათაური (სარჩევში ხვდება)
- `**გამუქებული აბზაცი**` → ლოცვა (სათაურს მიყოლებული ლოცვები იკეცება)
- `გუნდი:`, `დიაკონი:`, `მღვდელი:`, `მღვდელმთავარი:`, `მკითხველი:` … → მთქმელის
  სტრიქონი (ფერი/ნიშანი თავად მიენიჭება)
- ჩვეულებრივი აბზაცი → განგების მითითება (რუბრიკა, წითელი დახრილი)
- `*სიტყვა*` → გამოყოფა ტექსტში

## რეჟიმები (mode)

- *(არაფერი)* — საღმრთო მსახურების ტექსტი: ჩვეულებრივი აბზაცი = რუბრიკა
- `hybrid` — შერეული: აბზაცი რუბრიკაა მხოლოდ აშკარა ნიშნებით (მოქმედი პირი,
  მოქმედების ზმნა, ფრჩხილები), დანარჩენი — საკითხავი ტექსტი
- `text` — საკითხავი: აბზაცები ტექსტია, მოკლე უსასვენო სტრიქონები — სათაურები

## შენიშვნა

გენერირებული JSON (`public/data/`) git-ში არ ინახება — ის ყოველ build-ზე
ამ ფაილებიდან იქმნება. წყარო მხოლოდ ეს საქაღალდეა.
