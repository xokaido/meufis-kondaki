# მეუფის კონდაკი — მობილური წიგნი

მობილურზე მორგებული ვებ-აპლიკაცია (PWA) მღვდელმთავრის მსახურებების თვალყურის დევნებისთვის.

**ცოცხალი ვერსია: https://kondaki.ge** (სარეზერვო: https://xokaido.github.io/meufis-kondaki/)

ჰოსტინგი (ორი დამოუკიდებელი არხი):

- **მთავარი — Cloudflare Workers** (https://kondaki.ge): Cloudflare-ის საკუთარი
  git-ინტეგრაცია ყოველ `main`-ზე push-ისას უშვებს `npm ci && npm run check &&
  npm test && npm run build`-ს (კონფიგი — `wrangler.jsonc`) — ტესტების ჩავარდნა
  წყვეტს გამოქვეყნებას. ეს ამ რეპოს workflow-ების გარეთ ხდება, Cloudflare-ის მხარეს.
- **სარეზერვო — GitHub Pages** (https://xokaido.github.io/meufis-kondaki/):
  `.github/workflows/deploy.yml` ჯერ ამოწმებს პროექტს (svelte-check, vitest,
  Playwright) და მხოლოდ წარმატების შემდეგ აქვეყნებს `dist/`-ს `gh-pages`-ზე.

## გაშვება

საჭიროა Node.js **20.19+** (რეკომენდებულია 22 — იხ. `.nvmrc`).

**დეველოპმენტი:**

```sh
npm install && npm run dev
```

გახსენით: `http://localhost:5173`

**სამუშაო build:**

```sh
npm run build
```

გამოაქვს `dist/` საქაღალდე — ატვირთეთ ნებისმიერ სტატიკურ ჰოსტინგზე (Cloudflare, GitHub Pages, Netlify და ა.შ.).

## შესაძლებლობები

- სამი მსახურება საწყის ეკრანზე, წაკითხვის პროგრესით
- ფერებითა და ნიშნებით გამოყოფილი მთქმელები: მღვდელმთავარი (☩ ჯვარი), მღვდელი (ჯვარი), დიაკონი (საცეცხლური), გუნდი (ნოტები), მედავითნე/მუხლი (წიგნი)
- ლოცვების ჯგუფები (ლოცვა ა, ბ, გ…) — ჩაკეცვა/გაშლა სათაურზე შეხებით
- განგების მითითებები (რუბრიკები) — დახრილი, ჩაქრობადი ღილაკით
- სარჩევი ☰ — სწრაფი გადასვლა მსახურების ნაწილებზე
- შრიფტის ზომა A−/A+, ბნელი/ნათელი თემა
- „⏾ ეკრანი" — ეკრანი არ ჩაქრება მსახურების დროს (Wake Lock)
- პოზიცია ინახება — გახსნისას იქ დაბრუნდებით, სადაც გაჩერდით

## სტრუქტურა

- `texts/` — მსახურებების საწყისი ტექსტები: თითო ფაილი = თითო ტექსტი,
  ზევით frontmatter-ით (სათაური, კატეგორია, სარჩევის ანკერები).
  **ახალი ტექსტის დასამატებლად მხოლოდ აქ ფაილის ჩამატებაა საჭირო** —
  ფორმატი აღწერილია `texts/README.md`-ში
- `build.cjs` — გადამყვანი: `texts/*.md` → `public/data/*.json` + ძიების ინდექსი (`npm run data`)
- `src/` — Svelte 5 აპლიკაცია (views/, components/, lib/, styles/)
- `public/` — სტატიკური ფაილები (fonts/, icons/; `public/data/` — build.cjs-ის გამოყვანა)
- `tests/` — ერთეული ტესტები (vitest) + სმოუქ ტესტები (Playwright, `tests/e2e/`)
- `docs/` — სპეციფიკაციები, რევიუები და არქივი (`docs/archive/sources/` —
  თავდაპირველი გაერთიანებული ფაილები და ერთჯერადი გადამყვანი სკრიპტი)
- `dist/` — build-ის შედეგი (vite build, არ ინახება git-ში)

ტექსტის შესწორების შემდეგ ხელახლა გაუშვით `npm run build` — ის თავად
განაახლებს მონაცემთა ფაილებს და ქეშის მანიფესტს, ასე რომ დაყენებული
აპლიკაცია შემდეგ გახსნაზე ახალ ტექსტს ჩამოტვირთავს.

## ტექსტში შეცდომის გასწორება (ბრაუზერიდან, დაყენების გარეშე)

1. გახსენით https://github.com/xokaido/meufis-kondaki/tree/main/texts
2. დააწკაპუნეთ სასურველ ფაილზე (მაგ. `10-vespers.md` — მწუხრი; სრული
   ცხრილი იხ. იქვე README-ში)
3. ზედა მარჯვნივ აირჩიეთ ფანქრის ✏️ ნიშანი (Edit this file)
4. გაასწორეთ ტექსტი და ზევით მარჯვნივ დააჭირეთ **Commit changes** → კიდევ ერთხელ **Commit changes**
5. დაახლოებით 1–2 წუთში საიტი თავად განახლდება; დაყენებული აპლიკაცია
   ახალ ტექსტს მომდევნო გახსნაზე ჩამოტვირთავს

ფაილების ფორმატი: **გამუქებული** მოკლე სტრიქონი = სათაური, გამუქებული
აბზაცი = ლოცვა, „გუნდი:", „დიაკონი:" და მისთ. = მთქმელი, ჩვეულებრივი
აბზაცი = განგების მითითება. ტექსტში სიტყვის გამოსაყოფად შემოსაზღვრეთ
ერთი ვარსკვლავით: `*შიოსათვის*` — გამოჩნდება გამუქებული, ოქროსფერი.

## განახლების გამოქვეყნება (ავტომატური)

ყოველი push `main`-ზე ორივე არხი თავად განახლდება: Cloudflare აშენებს და
აქვეყნებს https://kondaki.ge -ს, ხოლო GitHub Actions ტესტების გავლის შემდეგ
ანახლებს სარეზერვო GitHub Pages-ს (`.github/workflows/deploy.yml`).
ლოკალურად საკმარისია:

```sh
git add -A && git commit -m "..." && git push origin main
```
